# ゲームエンジンの設計と処理フロー

本ゲームは、React の宣言的な UI と、Canvas による命令的なゲームループを組み合わせた「ハイブリッド設計」を採用しています。

## 1. フォルダ構成と責務の分離
プロジェクトは「関心の分離」に基づき、機能ごとに細分化されています。

- **`src/types/game.ts`**
    - 役割: ステージやユニットのデータ構造（型）の定義。
- **`src/constants/units.ts`**
    - 役割: 味方・敵ユニットのマスターデータ。
- **`src/constants/stages/`**
    - 役割: ステージごとの設定（背景色、難易度、拠点HPなど）。
    - `index.ts`: 全ステージの目録。
    - `stage1.ts`, `stage2.ts`: 個別のステージデータ。
- **`src/hooks/useGameAudio.ts`**
    - 役割: Web Audio API によるサウンド管理。
- **`src/components/Canvas/`**
    - 役割: 描画エンジン。
    - `renderer.ts`: 全体描画の司令塔。
    - `units/`: キャラクター個別の描画ロジック。
    - `stage/`: 背景・城などの環境描画ロジック。
- **`src/components/Game.tsx`**
    - 役割: メインループの管理、物理演算、UI統合。

## 2. データ駆動型ステージシステム
新しいステージを追加する際、プログラム本体（Game.tsx）を修正する必要はありません。
`src/constants/stages/` に新しい定義ファイルを追加し、`index.ts` の目録に登録するだけで、ステージ選択画面に自動的に反映されます。

## 3. 描画パイプライン
描画処理は以下のステップで実行されます。
1.  **ステージ背景:** `drawBackground` が現在のステージ設定に応じた色で空や地面を描画。
2.  **拠点:** `drawCastle` が城を描画。
3.  **ユニット:** 陣営に応じて `drawCat` または `drawDog` を呼び出し。
4.  **UI:** お金、レベル、HPゲージを描画。

## 4. 安定した計算エンジン
`useRef` (stateRef) を使用することで、React のライフサイクルから独立した 60fps の計算を保証。`deltaTime` による正規化により、動作環境に左右されない一定のゲームスピードを実現しています。
