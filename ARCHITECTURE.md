# ゲームエンジンの設計と処理フロー

本ゲームは、React の宣言的な UI と、Canvas による命令的なゲームループを組み合わせた「ハイブリッド設計」を採用しています。

## 1. 状態管理の二重構造
本プロトタイプ最大の特徴は、状態管理を「ロジック用」と「表示用」で分けている点です。

- **stateRef (useRef):** 
    - 役割: 60fps で動く物理演算や数値計算の「真の実体」。
    - 特徴: 値を書き換えても React の再レンダリングが発生しないため、爆速で計算が可能です。
- **ui (useState):**
    - 役割: 画面下のボタンや、Canvas 外のテキスト表示用。
    - 特徴: 10フレームに1回（約0.16秒ごと）だけ `stateRef` の値をコピーして更新します。これにより、React の負荷を最小限に抑えています。

## 2. メインループのフロー
`requestAnimationFrame` を使用し、以下の順序で処理が実行されます。

### ① 時間の正規化 (deltaTime)
前回のフレームからの経過時間（dt）を計算します。これにより、マシンの性能に関わらず、ユニットの移動速度やダメージ速度が一定に保たれます。

### ② 更新処理 (update関数)
1.  **経済計算:** 働きネコのLvに応じてお金を増加させます。
2.  **にゃんこ砲チャージ:** ゲージを一定割合で加算します。
3.  **クールタイム消費:** 出撃ボタンの待ち時間を減らします。リキャが完了した瞬間に「チャリン」音を鳴らします。
4.  **敵の自動生成:** タイマーが一定値を超えると、敵（わんこ）を配列に追加します。
5.  **戦闘演算 (最重要):** 
    - 全ユニットをループで回し、射程内に敵がいるかチェック。
    - 敵がいれば移動を停止し、相手のHPを削る（`damage * dt`）。
    - 城の前に到達していれば、城のHPを削る。
    - HPが0以下のユニットを配列から除外。

### ③ 描画処理 (draw関数)
Canvas 上に以下のレイヤー順で上書き描画します。
1.  **背景:** 空（グラデーション）、山、流れる雲。
2.  **地面:** 芝生と土。
3.  **城:** 味方と敵の城、およびその上のHPバー。
4.  **ユニット:** `drawUnit` 関数を呼び出し、猫や犬を描画。`Math.sin` を使ってピョコピョコ跳ねるアニメーションを適用。
5.  **エフェクト:** にゃんこ砲発射時は白いレーザーを描画。
6.  **UIオーバーレイ:** 左上のお金とレベルのボックス。

## 3. 戦闘ダメージの計算
本ゲームでは「自分が受けるダメージ」を毎フレーム合算する方式をとっています。
```typescript
// 他の全てのユニットをチェック
currentUnits.forEach(other => {
  if (other.type !== unit.type) { // 相手が敵軍なら
    const dist = Math.abs(other.x - unit.x);
    if (dist < other.stats.range) { // 相手の射程内に自分がいれば
      damageTaken += other.stats.damage * dt; // ダメージを受ける
    }
  }
});
```
この方式により、ユニットがどれだけ重なっていても（本家スタイル）、正確に集団戦のダメージを計算できます。

## 4. サウンド合成
`Web Audio API` を使用し、以下の波形を組み合わせて音を作っています。
- **サイン波 (Sine):** 澄んだ音。召喚音（ピコッ）や完了音（チャリン）に使用。
- **三角波 (Triangle):** 少し柔らかい音。BGMのメロディに使用。
- **矩形波 (Square):** ファミコン風の音。攻撃音（ガシッ）の核となる衝撃に使用。
- **鋸歯状波 (Sawtooth):** ザラザラした音。にゃんこ砲の充填音や敗北ジングルに使用。
