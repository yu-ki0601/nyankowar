# ゲームエンジンの設計と処理フロー

本ゲームは、React の宣言的な UI と、Canvas による命令的なゲームループを組み合わせた「ハイブリッド設計」を採用しています。

## 1. 状態管理とレンダリングの最適化
- **stateRef (useRef):** 全ての物理演算、HP管理、移動、タイマーを 60fps で計算。React の再レンダリングを介さないため、極めて高いパフォーマンスを発揮します。
- **ui (useState):** 必要な情報のみを stateRef からコピーし、UIパーツ（ボタンの有効化、数値表示）のみを更新。

## 2. 強化版音声システム (`useGameAudio.ts`)
ブラウザの音声再生制限と安定性を両立するため、以下の設計を採用しています。
- **シングルトン・コンテキスト:** `AudioContext` をブラウザ全体で1つに固定し、再読み込みやレンダリングの影響を排除。
- **再帰的スケジューラ:** `setInterval` の代わりに `setTimeout` の再帰呼び出しを使用。ブラウザのバックグラウンド動作時でもリズムが崩れず、安定したBGM再生を実現。
- **即時レスポンス:** 効果音（ガシッ、チャリン等）は同期的にオシレーターを生成。遅延のない手応えのあるサウンドを提供します。

## 3. ステージ拡張システム (`src/constants/stages/`)
- **完全分離:** ステージごとの固有設定（色彩、敵の強さ、湧き速度）を独立したファイルに集約。
- **目録管理:** `index.ts` に登録するだけで、メインメニューに新しい章が自動追加されるプラグインライクな設計。

## 4. 描画パイプライン (`src/components/Canvas/`)
描画の責務を最小単位に分割しています。
- `stage/`: 環境、城。
- `units/`: 猫、犬（わんこ）。
- `renderer.ts`: 各レイヤーの重ね合わせ順序を制御。

## 5. 戦闘ロジック
- **範囲ダメージ:** 射程内にいる「全ての敵」にダメージを蓄積させる計算方式。これにより、ユニットが重なり合って戦う本家のプレイ感を数学的に再現しています。
- **ノックバック:** にゃんこ砲発射時に、座標値を直接加算することで敵軍を物理的に押し戻します。
